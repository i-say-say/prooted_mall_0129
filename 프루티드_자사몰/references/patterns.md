# 레퍼런스 브랜드 공통 IA 패턴

## 분석 대상 브랜드 (8개)

| 브랜드 | 카테고리 | 특징 |
|--------|----------|------|
| Growus | 헤어케어 | 심플한 구조, 자연 성분 강조 |
| DR.FORHAIR | 헤어/두피케어 | 두피고민별 분류, 전문성 강조 |
| NARKA | 헤어 스타일링 | 겟더룩(스타일링), 트렌디한 UX |
| ViveLab | 두피케어 | 라인별 세분화, 솔루션 시스템 |
| Refilled | 탈모케어 | AI 진단 도구, 데이터 기반 |
| PO:DL | 스킨케어 | 슈퍼푸드 스토리텔링 |
| LILYEVE | 스킨케어 | 핵심 기술 강조, 오프라인 연계 |
| HAR | 헤어케어 | 다국어 지원, 글로벌 타겟 |

---

## 1. 헤더 네비게이션 공통 구조

### 필수 메뉴 (모든 브랜드 공통)
- **SHOP** - 상품 카테고리 진입점
- **BRAND/ABOUT** - 브랜드 스토리, 철학
- **CUSTOMER/COMMUNITY** - 고객지원 (FAQ, 공지, 문의)

### 선택 메뉴 (브랜드별 차별화)
- EVENT/PROMOTION - 이벤트, 프로모션
- MEMBERSHIP - 멤버십/등급 혜택
- REVIEW - 리뷰 별도 메뉴화

### 유틸리티 영역 (우측 상단)
- 검색
- 로그인/회원가입
- 마이페이지
- 장바구니

---

## 2. 상품 카테고리 구조 패턴

### A. 기본 분류
```
전체상품
├── BEST (베스트셀러)
├── NEW (신상품)
└── SALE (할인/기획)
```

### B. 유형별 분류 (헤어케어 기준)
```
유형별
├── 샴푸
├── 트리트먼트/린스
├── 헤어에센스/오일
├── 스타일링
└── 두피케어
```

### C. 고민별 분류 (DR.FORHAIR, ViveLab 참고)
```
고민별
├── 탈모/모발 강화
├── 손상모/데미지 케어
├── 두피 트러블
├── 건조/보습
└── 볼륨/스타일링
```

### D. 세트/기획
```
기획상품
├── 베스트 세트
├── 트라이얼 키트
└── 시즌 한정
```

---

## 3. 메인 페이지 구성 패턴

### 공통 섹션 (상단→하단 순서)
1. **히어로 배너** - 슬라이드 형식 (3~8개)
2. **BEST PRODUCTS** - 인기상품 4~8개
3. **NEW ARRIVALS** - 신상품 소개
4. **브랜드 스토리 존** - 철학, 성분, 기술
5. **REVIEW 섹션** - 고객 후기 슬라이더
6. **프로모션/이벤트 배너**

### 차별화 섹션 (선택)
- 고민별 추천 (DR.FORHAIR)
- 진단 도구 연결 (Refilled)
- 라인업 탭 (ViveLab)
- 스토리 콘텐츠 (PO:DL)

---

## 4. 전환 구조 (Conversion Flow)

### 기본 구매 흐름
```
메인 → 카테고리 → 상품상세(PDP) → 장바구니 → 결제
```

### 진단형 구매 흐름 (Refilled 참고)
```
진단 도구 → 맞춤 추천 → 상품상세 → 장바구니 → 결제
```

### 멤버십 전환 흐름
```
비회원 → 회원가입(첫구매 혜택) → 첫 구매 → 재구매 유도
```

---

## 5. 푸터 구조 패턴

### 좌측 영역
- 회사 정보 (상호, 대표, 사업자번호, 주소)
- 통신판매신고번호

### 중앙 영역
- 고객센터 (전화, 운영시간)
- 이메일 문의
- 카카오톡 상담

### 우측 영역
- 이용약관
- 개인정보처리방침
- SNS 링크 (Instagram, Kakao)

---

## 6. 프루티드 적용 시 참고 포인트

### 강점 활용
- **슈퍼푸드 성분** → PO:DL의 "슈퍼푸드 스토리" 형식 참고
- **자연 유래** → Growus의 클린 뷰티 톤앤매너 참고
- **전문성** → DR.FORHAIR의 고민별 분류 참고

### 차별화 기회
- 성분 스토리텔링 강화 (슈퍼푸드 별 페이지)
- 간단한 모발 진단 도구 도입 가능성
- 비건/클린 뷰티 인증 강조
